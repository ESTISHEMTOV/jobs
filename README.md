# לוח משרות מערכות מידע — מדריך תחזוקה 🛠️

עמוד המשרות מתעדכן אוטומטית כל בוקר, **כולו רץ בתוך GitHub שלך** — בלי תלות בשום חשבון חיצוני.

**הכתובת הציבורית:** https://estishemtov.github.io/jobs/

## איך זה עובד
- כל בוקר ב-07:00 (שעון ישראל) רץ אוטומטית "GitHub Action".
- הוא מריץ סקריפט שמחפש משרות, ומבקש ממודל בינה (Gemini, חינמי) לסנן ולסדר אותן.
- התוצאה נשמרת לעמוד, ו-GitHub Pages מציג אותו.

## הקבצים (ומה מותר לערוך)
| קובץ | מה זה | לערוך? |
|---|---|---|
| `scripts/refresh.mjs` | **המוח** — כל המקורות והכללים | ✅ כן |
| `.github/workflows/daily-jobs.yml` | התזמון (שעה) | ✅ רק את השעה |
| `index.html` | העמוד עצמו | ❌ נוצר אוטומטית — לא לגעת |
| `data/*.json` | זיכרון "הוצג אתמול" | ❌ לא לגעת |

## הדרך הכי קלה לשנות (מומלץ למי שלא טכני/ת)
1. פותחים את הקובץ `scripts/refresh.mjs` ב-GitHub, מעתיקים את **כל** התוכן.
2. מדביקים לכל עוזר AI שיש לך (ChatGPT / Gemini / Claude / כל אחד) וכותבים מה רוצים, למשל:
   *"זה סקריפט שאוסף משרות. תוסיף לי את האתר X כמקור נוסף, ותחזיר לי את הקובץ המלא."*
3. מעתיקים את הקובץ שחזר, חוזרים ל-GitHub, לוחצים על העיפרון ✏️ בקובץ `scripts/refresh.mjs`, מוחקים הכול, מדביקים את החדש, ו-**Commit changes**.

## להוסיף אתר מקור — ידנית (אם רוצים)
בראש `scripts/refresh.mjs` יש רשימה בשם `BOARDS`. כל שורה = `['שם האתר', 'כתובת עמוד חיפוש']`. להוספת אתר, מוסיפים שורה:
```js
const BOARDS = [
  ['JobMaster', 'https://www.jobmaster.co.il/jobs/?q=מנהל מערכות מידע'],
  ['AllJobs', 'https://www.alljobs.co.il/SearchResultsGuest.aspx?freetxt=מנהל מערכות מידע'],
  // ↓ הוסיפי שורה כזו עם האתר החדש:
  ['שם-האתר-החדש', 'https://כתובת-החיפוש-באתר/?q=מנהל מערכות מידע'],
];
```
שומרים (Commit) — וזהו. בהרצה הבאה האתר ייכלל.

## פעולות נפוצות
- **לשנות את שעת הריצה:** בקובץ `.github/workflows/daily-jobs.yml`, השורה `cron: '0 4 * * *'` היא בשעון UTC (04:00 UTC = 07:00 בישראל). לשעה 08:00 בישראל → `'0 5 * * *'`.
- **להריץ עכשיו (ידנית):** לשונית **Actions** → **Daily Jobs Refresh** → **Run workflow**.
- **להחליף את מפתח ה-Gemini:** מפתח חדש חינמי ב-https://aistudio.google.com/apikey ← ואז Settings → Secrets and variables → Actions → `GEMINI_API_KEY` → Update.
- **לראות אם הריצה הצליחה:** לשונית **Actions** — וי ירוק = הצליח.

## תקלות נפוצות
- **העמוד לא התעדכן:** בדקי ב-Actions אם הריצה אדומה; לרוב זה מפתח Gemini שפג/חרג ממכסה → צרי מפתח חדש ועדכני את ה-Secret.
- **הדפדפן מציג גרסה ישנה:** רענון קשיח (Ctrl+F5).
